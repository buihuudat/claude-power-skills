# /explain — Deep Code Explanation

Explain: $ARGUMENTS

---

## 1. Big Picture (understandable in 30 seconds)
- What is this, what does it do, why does it exist
- Its position in the larger system

## 2. How It Works (mental model)
- Data flow: how data enters and exits
- Control flow: where decisions are made
- State: what changes, what does not change
- Side effects: does it affect anything outside its scope

## 3. Key Design Decisions
- Why was it written this way (not another way)?
- Tradeoffs that were accepted
- What constraints shaped this design

## 4. Non-Obvious Parts
- Code that looks strange but has a specific reason
- Hidden bugs or gotchas to be aware of
- Important assumptions

## 5. How to Use / Modify Safely
- The correct way to use it
- Things NOT to do when modifying it
- Extension points if new functionality is needed

---
Explain at the level appropriate to the context; use analogies when concepts are abstract.
